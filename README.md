**How can I ensure that my code, program, or software is functional and secure?

Ensuring my code is functional and secure comes down to a couple topics. The first and most important is following safe coding procedures and industry standards. These standards are set to help developers write clean and precise code. Validating and sanitizing all inputs, encrypting sensitive data, along with a plethora of other techniques are all used to ensure the code is secure and clean. Another method using correct variable and method names while keeping functions small and singular topic focused. The last is preforming comprehensive testing on the code. This ensure that what I have written works well and contains no defects.

**How do I interpret user needs and incorporate them into a program

Interpreting user needs comes from the requirements gathering stage. This is when the team learns exactly what the client is hoping for and can then begin writing code and programs. Constant communication between client and team is crucial in this way because if something happens during coding and the team is unable to meet a requirement we can speak with the client and see what the next best options are to ensure we are still delivering a program that will satisfy the customer. 

**How do I approach designing software?

I approach designing software as analytical as I can. I do not dive straight into coding but first map out what needs to be done and how I can accomplish that. I find it easier to break large programs into smaller sections and integrate them together. This allows me to put my focus on one area and give it my best work instead of constantly switching tasks and dividing my attention. I also am aware that as a beginning developer I do not have all the skills I wish I had so I am not afraid to research and ask for help when I need it. I refuse to let me ego stand in the way of putting my best work forward.
